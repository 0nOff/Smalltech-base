# Conference Program Guide

Коллаборативный исследовательский проект о smalltech/midtech-компаниях (малых и средних технологических командах) и их инженерных практиках.

Сайт в этом репозитории - это оболочка для публикации и обсуждения выводов. Главная ценность проекта - не только интерфейс, а аргументированные материалы, комментарии и контраргументы сообщества.

## Что это за проект

- Проект собирает и структурирует наблюдения о том, как small/midtech-команды строят разработку, процессы и архитектуру под ограничениями.
- Базовые находки уже представлены в виде интерактивной карты треков.
- Репозиторий открыт для профессиональной дискуссии и дополнения исходных тезисов.

## Для чего нужна коллаборация

- Дополнять проект новыми текстовыми материалами.
- Оспаривать или уточнять текущие выводы проекта.
- Привносить альтернативные интерпретации данных и практический опыт из команд.

## Формат вкладов

Мы приглашаем контрибьюторов в первую очередь к **некодовым** изменениям:

- аналитические заметки и мини-статьи;
- комментарии к существующим тезисам;
- контраргументы к выводам проекта;
- кейсы из практики small/midtech-команд;
- уточнения терминов, формулировок и структуры аргументации.

## Как контрибьютить

1. Сделайте fork репозитория и создайте ветку под ваш материал.
2. Добавьте текстовый материал в `README.md` или отдельный markdown-файл (рекомендуется для больших текстов).
3. Если вы пишете возражение или контраргумент, явно укажите:
   - с каким тезисом вы не согласны;
   - на каких наблюдениях/данных строится ваша позиция;
   - какие практические последствия следуют из вашей позиции.
4. Откройте Pull Request с кратким описанием:
   - **тип вклада**: комментарий / контраргумент / дополнение / кейс;
   - **суть тезиса** в 3-5 пунктах;
   - **контекст**: роль, тип компании, стадия, ограничения (по желанию).

## Шаблон Pull Request для текстовых вкладов

Используйте такую структуру описания PR:

```md
## Тип вклада
Комментарий / Контраргумент / Дополнение / Кейс

## К какому тезису относится
Ссылка на раздел и короткая формулировка исходного тезиса.

## Основная позиция
1. ...
2. ...
3. ...

## Основания
- Практический опыт / данные / наблюдения.
- Ограничения, при которых вывод применим.

## Что это меняет в проекте
- Какие формулировки или выводы стоит пересмотреть.
```

## Правила обсуждения

- Критика тезисов приветствуется, переход на личности - нет.
- Предпочтительны проверяемые аргументы и конкретные примеры.
- Если позиция спорная, фиксируйте границы применимости (контекст компании, команды, домена).
- Мы не стремимся к единственной "правильной" модели: важна карта обоснованных позиций.

## Принцип развития проекта

Проект развивается как открытая редакционная площадка: сначала появляется тезис, затем вокруг него накапливаются подтверждения, возражения, уточнения и практические кейсы. Pull Request в этом процессе - это единица осмысленной дискуссии, а не только способ изменить код.
